# PDF Receipt Generation — Sub-Plan

> Target: workflows that reach `approved` status get a downloadable PDF receipt for compliance and audit trails.
> Linked from: `docs/plans/2026-05-17-workflow-engine-roadmap.md`

## Goal

Let users download a formal PDF receipt for any fully-approved workflow. Useful for Islamic finance compliance, zakat documentation, mosque expense records, and audit trails.

## Data to include on the receipt

- Organization name
- Workflow title + description
- Template name
- Submission date
- Final approval date
- List of all approval steps with: step name, approver name, approval date, note (if any)
- Unique receipt ID (workflow ID)
- HalalFlow branding (logo text, URL)

## Library choice

**`pdf-lib`** — pure JavaScript, no native dependencies, runs in Node.js (Next.js API routes), supports text layout and basic drawing. Alternatives considered:
- `@react-pdf/renderer` — heavier, React-based, overkill for a server-side receipt
- `jspdf` — API is less ergonomic for multi-line text and page layout

## Implementation lanes

### Lane 1 — API skeleton
- Install `pdf-lib`
- Create `src/app/api/workflows/[id]/receipt/route.ts`
- GET handler: verify session, lookup workflow with approvals + approvers + org, return `application/pdf`
- For this lane: generate a minimal PDF with just the workflow title and "Receipt coming soon" text
- Set `Cache-Control: no-store`

### Lane 2 — Full PDF layout
- Design receipt layout in the API route:
  - Header: org name + "Workflow Receipt"
  - Body: workflow metadata block
  - Table-like approval chain (step #, name, approver, date, note)
  - Footer: generated by HalalFlow + timestamp + workflow ID
- Handle text wrapping for long descriptions/notes
- Error handling with `console.error` and 500 response

### Lane 3 — UI button
- Add a "Download receipt" button to `src/app/workflows/[id]/page.tsx`
- Only show when `workflow.status === 'approved'`
- Link to `/api/workflows/[id]/receipt` (opens in new tab or triggers download via `a[download]`)
- Use `FileArrowDown` Phosphor icon

### Lane 4 — Polish
- Add Arabic/RTL support if needed (future)
- Consider email attachment on final approval (future)

## Cycle protocol

1. Pick the first unchecked lane below.
2. Implement only that lane. Keep the change surgical.
3. Run `npm run lint` and `npm run build`.
4. If green, check the lane off and update this file + STATE.yaml.
5. Do not commit unless the lane explicitly says to.
